  <action>
    Create `app/plan/day/page.tsx` as a Server Component that queries today's data.

    Authentication: Use the standard pattern — `const supabase = await createClient()`, `supabase.auth.getUser()`, redirect to /auth if no user.

    Load user profile for timezone:
    ```
    const { data: profile } = await supabase
      .from("user_profiles")
      .select("timezone")
      .eq("id", user.id)
      .single();
    const timezone = (profile?.timezone as string | null) ?? "America/Boise";
    ```

    Compute today's boundaries using user timezone:
    - Import `startOfDay`, `endOfDay` from `date-fns` and `tz` from `@date-fns/tz`
    - `const todayStart = startOfDay(new Date(), { in: tz(timezone) })`
    - `const todayEnd = endOfDay(new Date(), { in: tz(timezone) })`

    Query today's plan_blocks:
    ```
    const { data: blocks } = await supabase
      .from("plan_blocks")
      .select("*, tasks(title, type, estimated_minutes, due_date, course_id, courses(name))")
      .eq("user_id", user.id)
      .gte("start_time", todayStart.toISOString())
      .lte("start_time", todayEnd.toISOString())
      .order("start_time", { ascending: true });
    ```

    Query top priority tasks (incomplete, ordered by due_date ASC, limit 5):
    ```
    const { data: priorityTasks } = await supabase
      .from("tasks")
      .select("id, title, type, due_date, estimated_minutes, status, course_id, courses(name)")
      .eq("user_id", user.id)
      .neq("status", "done")
      .order("due_date", { ascending: true, nullsFirst: false })
      .limit(5);
    ```

    Calculate estimated time remaining: sum minutes from blocks where `status === "scheduled"` (not done, not missed). For each block: `(end_time - start_time) / 60000`.

    Map blocks to the same shape used by PlanBlock component (id, start_time, end_time, status, tasks: { title, courses }).

    Render layout:
    - Import NavHeader from `@/app/plan/_components/NavHeader` (will exist from Plan 01, but if running in parallel, create a minimal fallback header)
    - Wrap in `<div className="min-h-screen bg-gray-50">`
    - Try to import NavHeader; if the import would fail at build because Plan 01 hasn't run yet, use the same inline header pattern from existing pages temporarily
    - `<main className="mx-auto max-w-5xl px-4 py-8">`
    - Pass blocks, priorityTasks, and remainingMinutes to DayTimeline client component

    Important: Use the same typing for blocks as PlanGrid does (PlanBlockRow shape).
  </action>
