  <action>
    Create four Supabase client utility files following the @supabase/ssr patterns:

    1. `lib/supabase/client.ts` — Browser client:
       - Export `createClient()` using `createBrowserClient` from `@supabase/ssr`
       - Uses `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY`

    2. `lib/supabase/server.ts` — Server client:
       - Export async `createClient()` using `createServerClient` from `@supabase/ssr`
       - Uses `cookies()` from `next/headers` for `getAll` and `setAll` cookie handlers
       - For use in Server Components, Server Actions, Route Handlers

    3. `lib/supabase/middleware.ts` — Middleware helper:
       - Export `updateSession(request: NextRequest)` function
       - Creates `createServerClient` with `request.cookies.getAll()` and custom `setAll` that writes to both `request.cookies` and `response.cookies`
       - Calls `supabase.auth.getUser()` to refresh tokens
       - Implements route protection logic:
         - Unauthenticated users accessing protected routes → redirect to `/auth`
         - Authenticated users accessing `/auth` → redirect to `/dashboard`
         - Public routes: `/auth`, `/auth/callback`
       - Returns the response with updated cookies

    4. `lib/supabase/admin.ts` — Admin client (for account deletion):
       - Export `createAdminClient()` using `createClient` from `@supabase/supabase-js` (NOT @supabase/ssr)
       - Uses `NEXT_PUBLIC_SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY`
       - This client bypasses RLS — ONLY use server-side for admin operations

    5. `middleware.ts` (project root):
       - Import `updateSession` from `lib/supabase/middleware`
       - Export middleware function that calls `updateSession`
       - Export config with matcher excluding static files: `/((?!_next/static|_next/image|favicon.ico|.*\\.(?:svg|png|jpg|jpeg|gif|webp)$).*)`
  </action>

<task type="auto">
  <name>Task 3: Create database migration with all tables, RLS policies, and storage bucket</name>
  <files>
    supabase/migrations/00001_initial_schema.sql
  </files>
  <action>
    Create `supabase/migrations/00001_initial_schema.sql` containing the complete schema:

    **Tables (all with UUIDs, TIMESTAMPTZ, cascade delete):**

    1. `terms` — id (UUID PK), user_id (UUID FK auth.users ON DELETE CASCADE), name (TEXT NOT NULL), start_date (DATE NOT NULL), end_date (DATE NOT NULL), created_at (TIMESTAMPTZ DEFAULT now()), updated_at (TIMESTAMPTZ DEFAULT now())

    2. `courses` — id (UUID PK), user_id (UUID FK auth.users ON DELETE CASCADE), term_id (UUID FK terms ON DELETE CASCADE), name (TEXT NOT NULL), meeting_times (JSONB), created_at, updated_at

    3. `tasks` — id (UUID PK), user_id (UUID FK auth.users ON DELETE CASCADE), course_id (UUID FK courses ON DELETE CASCADE), title (TEXT NOT NULL), type (TEXT NOT NULL CHECK IN ('assignment','exam','reading','other')), status (TEXT NOT NULL DEFAULT 'todo' CHECK IN ('todo','doing','done')), due_date (TIMESTAMPTZ), estimated_minutes (INTEGER), points (NUMERIC), weight (NUMERIC), needs_review (BOOLEAN DEFAULT false), created_at, updated_at

    4. `subtasks` — id (UUID PK), user_id (UUID FK auth.users ON DELETE CASCADE), task_id (UUID FK tasks ON DELETE CASCADE), title (TEXT NOT NULL), status (TEXT NOT NULL DEFAULT 'todo' CHECK IN ('todo','doing','done')), due_date (TIMESTAMPTZ), estimated_minutes (INTEGER), sort_order (INTEGER DEFAULT 0), created_at, updated_at

    5. `availability_rules` — id (UUID PK), user_id (UUID FK auth.users ON DELETE CASCADE), day_of_week (INTEGER NOT NULL CHECK 0-6), start_time (TIME NOT NULL), end_time (TIME NOT NULL), rule_type (TEXT NOT NULL CHECK IN ('available','blocked','preferred')), label (TEXT), created_at, updated_at

    6. `plan_blocks` — id (UUID PK), user_id (UUID FK auth.users ON DELETE CASCADE), task_id (UUID FK tasks ON DELETE SET NULL), start_time (TIMESTAMPTZ NOT NULL), end_time (TIMESTAMPTZ NOT NULL), status (TEXT NOT NULL DEFAULT 'scheduled' CHECK IN ('scheduled','done','missed')), created_at, updated_at

    **For each table:**
    - `ALTER TABLE public.{table} ENABLE ROW LEVEL SECURITY;`
    - Create 4 RLS policies (SELECT, INSERT, UPDATE, DELETE) using `auth.uid() = user_id`
    - Create index on `user_id` column: `CREATE INDEX idx_{table}_user_id ON public.{table}(user_id);`

    **Storage:**
    - Create `syllabi` bucket with `public = false`
    - RLS policies on `storage.objects` for syllabi bucket:
      - INSERT: `bucket_id = 'syllabi' AND auth.uid()::text = (storage.foldername(name))[1]`
      - SELECT: same condition
      - DELETE: same condition

    **Updated_at trigger:**
    Create a reusable function and trigger for `updated_at`:
    ```sql
    CREATE OR REPLACE FUNCTION update_updated_at()
    RETURNS TRIGGER AS $$
    BEGIN NEW.updated_at = now(); RETURN NEW; END;
    $$ LANGUAGE plpgsql;
    ```
    Apply trigger to all 6 tables.

    NOTE: This migration is designed to be run via the Supabase Dashboard SQL editor or `supabase db push`. The user will apply it manually as a checkpoint in Plan 04.
  </action>
