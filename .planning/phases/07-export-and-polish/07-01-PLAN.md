<task type="auto">
  <name>Task 1: Create .ics export API route</name>
  <files>app/api/plan/export/route.ts</files>
  <action>
Install ical-generator:
```bash
npm install ical-generator
```

Create GET route handler at app/api/plan/export/route.ts:

1. Authenticate user via createClient() + getUser(). Return 401 if not authenticated.
2. Query user_profiles for timezone (default "America/Boise").
3. Query plan_blocks with status "scheduled", joined with tasks(title, course_id, courses(name)), ordered by start_time ascending. Same join pattern as app/plan/page.tsx.
4. Create ical calendar using `ical-generator`:
   ```typescript
   import ical from "ical-generator";
   const calendar = ical({ name: "BlockPlan Study Schedule", timezone });
   ```
5. For each block, create an event:
   - start: new Date(block.start_time)
   - end: new Date(block.end_time)
   - summary: task title
   - description: course name (or empty string if no course)
   - timezone: user's timezone from profile
6. Return Response with:
   - body: calendar.toString()
   - Content-Type: "text/calendar; charset=utf-8"
   - Content-Disposition: 'attachment; filename="blockplan.ics"'

Import pattern: `import ical from "ical-generator";` (default import).
  </action>
  <verify>
Run `npx tsc --noEmit` to verify no type errors. Verify the route file exists and exports GET.
  </verify>
  <done>GET /api/plan/export returns a valid .ics file with Content-Type text/calendar and Content-Disposition attachment header. Each scheduled plan_block appears as a VEVENT with correct times and timezone.</done>
</task>
