1. Replace the existing placeholder `app/onboarding/page.tsx` with a Server Component that:
   - Authenticates with `getUser()` (not getSession) — redirect to `/auth` if no user
   - Queries Supabase for: term (terms table, user_id match), courses (if term exists), availability_rules (if user has any)
   - Computes `currentStep`: no term = 1, no courses = 2, no availability = 3, all exist = 4
   - If user already has term AND courses AND availability rules, redirect to `/dashboard` (returning user per CONTEXT.md)
   - Passes `initialStep`, `termId`, and any existing `courses` list to `WizardShell`

2. Create `app/onboarding/_components/WizardShell.tsx` ("use client"):
   - Receives `initialStep` prop and manages `currentStep` state
   - Renders a progress indicator (step dots or numbered steps: 1-Term, 2-Courses, 3-Availability, 4-Next Steps)
   - Renders the correct step component based on `currentStep`
   - Provides `onNext` callback to advance steps
   - Step 3 and 4 components will be created in Plan 02 — render placeholder text for those steps

3. Create `app/onboarding/_components/StepTerm.tsx` ("use client"):
   - Form with fields: name (text input), start_date (date input), end_date (date input)
   - Uses `useActionState(createTerm, initialState)` for form submission
   - Displays inline validation errors from Zod (field-level errors under each input)
   - Shows pending state on submit button ("Saving..." when pending)
   - On success (state.success = true), calls `onNext()` to advance to step 2

4. Create `app/onboarding/actions.ts` with `createTerm` Server Action:
   - "use server" directive
   - Authenticates with `getUser()`
   - Validates with `termSchema.safeParse()` — return `{ errors: flatten().fieldErrors }` on failure
   - Checks user doesn't already have a term (single term per user for MVP) — return error if term exists
   - Inserts into `terms` table with `user_id`, `name`, `start_date`, `end_date`
   - Calls `revalidatePath("/onboarding")` on success
   - Returns `{ success: true, termId: data.id }`

5. Update `app/dashboard/page.tsx` to check auth (getUser + redirect if not authenticated). Keep it as a placeholder but ensure it's a valid landing page for returning users.
