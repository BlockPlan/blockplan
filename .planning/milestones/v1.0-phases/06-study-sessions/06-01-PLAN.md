  <action>
**Create `lib/study/mock.ts`:**

Export a `getMockStudyAids()` function that returns a deterministic `StudyAids` object:
- `summary`: 5 generic bullet points about study material (e.g., "Key concept 1: Overview of the main topic and its significance")
- `keyTerms`: 3 sample term/definition pairs
- `questions`: 8 questions alternating between "recall" and "conceptual" types
- Content should be generic but realistic — clearly mock data, not tied to any specific subject

**Create `lib/study/generate.ts`:**

Follow the EXACT pattern from `lib/syllabus/parser-llm.ts`:

1. Import `{ generateText, Output, NoObjectGeneratedError }` from `"ai"`
2. Import `{ openai }` from `"@ai-sdk/openai"`
3. Import `{ studyAidsSchema, type StudyAids }` from `"./types"`
4. Import `{ getMockStudyAids }` from `"./mock"`

5. Set `MAX_NOTES_CHARS = 12_000`

6. Export async function `generateStudyAids(notes: string): Promise<{ data: StudyAids; isMock: boolean }>`:
   - Feature gate: if `!process.env.OPENAI_API_KEY`, return `{ data: getMockStudyAids(), isMock: true }`
   - Truncate notes to MAX_NOTES_CHARS
   - Call `generateText` with:
     - `model: openai("gpt-4o-mini")`
     - `experimental_output: Output.object({ schema: studyAidsSchema })`
     - `prompt`: Multi-line string with:
       - Role: "You are a study aid generator."
       - SAFETY GUARDRAIL: "Generate study aids only. Do not provide answers to graded assignments, write essays, or complete homework. Focus on comprehension, recall, and conceptual understanding."
       - Instructions: Generate 5-10 bullet summary, key terms with definitions, 8-12 practice questions (mix of recall and conceptual)
       - The truncated notes
   - Extract `experimental_output` from result
   - If null/undefined, return mock with isMock: true
   - On success, return `{ data: experimental_output, isMock: false }`
   - Catch `NoObjectGeneratedError` separately (warn, return mock)
   - Catch other errors (error log, return mock)

The `isMock` flag is critical — Plan 02's UI uses it to show a "Mock mode" banner.
  </action>
