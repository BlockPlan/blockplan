**Create `app/study/actions.ts`:**

"use server" module with a single exported function:

```
export async function generateStudyAidsAction(
  prevState: StudyState,
  formData: FormData
): Promise<StudyState>
```

Where `StudyState` is:
```typescript
export type StudyState = {
  error?: string;
  data?: StudyAids;
  isMock?: boolean;
  taskTitle?: string;
};
```

Implementation:
1. Get authenticated user via `createClient()` + `supabase.auth.getUser()` (same pattern as tasks/actions.ts)
2. Extract `taskId` and `notes` from formData
3. Validate `notes` is non-empty (return error if blank)
4. Query task from Supabase: `supabase.from("tasks").select("id, title, type").eq("id", taskId).eq("user_id", user.id).single()`
5. If task not found: return `{ error: "Task not found" }`
6. If task.type is NOT "exam" or "reading": return `{ error: "Study sessions are only available for exam and reading tasks. This helps you prepare for tests and understand readings — not complete graded assignments." }`
7. Call `generateStudyAids(notes)` from `lib/study/generate.ts`
8. Return `{ data: result.data, isMock: result.isMock, taskTitle: task.title }`

**Create `app/study/_components/StudySession.tsx`:**

"use client" component that receives `taskId: string` and `taskTitle: string` as props.

UI layout:
1. Header: "Study Session: {taskTitle}" with a back link to /tasks
2. Textarea (min 4 rows, full width) with placeholder "Paste your notes or chapter headings here..."
3. Hidden input for taskId
4. "Generate Study Aids" button (disabled while pending)
5. When state has data, show three collapsible sections:

   **Summary** — Unordered bullet list of summary points
   **Key Terms** — Table or definition list with term in bold and definition
   **Practice Questions** — Numbered list with question text and a small badge showing "recall" or "conceptual"

6. If `state.isMock` is true, show a banner at top of results: "Mock mode — configure OpenAI API key for real study aids" (yellow/amber background, informational)
7. If `state.error`, show error message in red
8. "Regenerate" capability: user can edit the textarea and click Generate again — the form just resubmits

Use `useActionState` pattern (same as TaskForm):
```
const [state, formAction, isPending] = useActionState(generateStudyAidsAction, {});
```

Style with Tailwind — clean, readable layout. Each output section should have a heading and be visually distinct. Use card-like containers (rounded, border, padding) for each section.

**Create `app/study/page.tsx`:**

Server Component:
1. Read `searchParams` to get `task_id`
2. If no task_id, redirect to /tasks
3. Query the task from Supabase to get its title and type (for display)
4. If task not found, redirect to /tasks
5. If task.type is not "exam" or "reading", render an error message: "Study sessions are available for exam and reading tasks only" with a link back to /tasks
6. Render `<StudySession taskId={task.id} taskTitle={task.title} />`
