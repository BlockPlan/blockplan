<feature>
  <name>Pure Scheduler: Greedy EDD Bin Packing</name>
  <files>lib/services/scheduler.ts, lib/services/__tests__/scheduler.test.ts</files>
  <behavior>
The scheduler takes tasks, availability rules, planner settings, user timezone, and a plan start time, then produces scheduled blocks.

**Install dependencies first:** `npm install @date-fns/tz`

**Type definitions to export from scheduler.ts:**
- AvailabilityRule: { day_of_week: number (0-6), start_time: string ("HH:MM"), end_time: string ("HH:MM"), rule_type: 'available' | 'blocked' | 'preferred' }
- SchedulableTask: { id: string, title: string, course_id: string, due_date: string | null (ISO), estimated_minutes: number, status: 'todo' | 'doing' }
- PlannerSettings: { max_block_minutes: number, min_block_minutes: number, buffer_minutes: number }
- ScheduledBlock: { task_id: string, start_time: Date (UTC), end_time: Date (UTC) }
- SchedulerResult: { blocks: ScheduledBlock[], unscheduled: SchedulableTask[], risk_tasks: Array<{ task: SchedulableTask, level: 'at_risk' | 'overdue_risk' }> }

**Test cases (RED phase — write these first):**

1. **Empty tasks returns empty result:** generateSchedule([], rules, settings, tz, now) => { blocks: [], unscheduled: [], risk_tasks: [] }

2. **Empty availability returns all tasks as unscheduled:** generateSchedule(tasks, [], settings, tz, now) => { blocks: [], unscheduled: tasks, risk_tasks: [...] }

3. **Single task fits in single window:** Task with 45 min, one available window of 2 hours => 1 block of 45 min

4. **Task split across blocks:** Task with 150 min, max_block=90 => 2 blocks (90 + 60 min)

5. **Buffer between blocks:** Two tasks, 30 min each, buffer=10 => blocks separated by 10 min gap

6. **EDD priority:** Task A due April 5, Task B due April 3 => Task B scheduled first

7. **Null due date tasks scheduled last:** Task A has no due_date, Task B due April 5 => Task B first, then Task A

8. **Min block length respected:** Task with 10 min remaining, min_block=25, window has 20 min left => not placed (too small)

9. **Overdue risk detection:** Task due in the past => appears in risk_tasks as 'overdue_risk'

10. **At risk detection:** Task with more estimated_minutes than available windows before due date => appears in risk_tasks as 'at_risk'

11. **Windows in the past are skipped:** planStart is midday, morning windows skipped

12. **Partially elapsed window is trimmed:** planStart is 10:30, window starts at 9:00, scheduler starts at 10:30 not 9:00

**Algorithm (GREEN phase — implement after tests):**

1. Sort tasks: due_date ASC, null due_dates last
2. Build remaining-minutes map for each task
3. Build 7-day availability windows using @date-fns/tz:
   - For each day (0-6 from planStart), find availability_rules matching that day_of_week with rule_type='available'
   - Convert wall-clock times to UTC using TZDate/tz() context from @date-fns/tz
   - Skip windows ending before planStart; trim windows starting before planStart
   - Sort windows by start time ascending
4. Cursor walks through windows:
   - Find highest-priority task with remaining minutes > 0
   - Calculate block duration: min(remaining, max_block_minutes, window_available)
   - If duration < min_block_minutes, skip this window slot
   - Create ScheduledBlock, deduct from remaining, advance cursor by duration + buffer
5. After all windows processed:
   - Tasks with remaining > 0 go to unscheduled
   - Classify risk: overdue_risk if due_date <= now, at_risk if insufficient time before due_date

**IMPORTANT implementation detail:** Use `{ in: tz(userTimezone) }` context option from @date-fns/tz for all date-fns functions that need timezone awareness (startOfDay, getDay, addDays). This is the date-fns v4 way. Do NOT use the old date-fns-tz package.
  </behavior>
