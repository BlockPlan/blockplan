  <files>supabase/migrations/00002_user_profiles.sql, lib/validations/planner.ts</files>
  <action>
Create migration `supabase/migrations/00002_user_profiles.sql`:
- CREATE TABLE public.user_profiles with columns:
  - id UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE
  - timezone TEXT NOT NULL DEFAULT 'America/Boise'
  - planner_settings JSONB NOT NULL DEFAULT '{"max_block_minutes": 90, "min_block_minutes": 25, "buffer_minutes": 10}'::jsonb
  - created_at TIMESTAMPTZ DEFAULT now() NOT NULL
  - updated_at TIMESTAMPTZ DEFAULT now() NOT NULL
- Enable RLS with 3 policies: SELECT, INSERT, UPDATE (all using auth.uid() = id). No DELETE policy — profiles cascade from auth.users deletion.
- Add update_updated_at trigger (function already exists from 00001).

Create `lib/validations/planner.ts`:
- Export `plannerSettingsSchema` using Zod v3:
  - max_block_minutes: z.number().int().min(25).max(120).default(90)
  - min_block_minutes: z.number().int().min(15).max(60).default(25)
  - buffer_minutes: z.number().int().min(0).max(30).default(10)
- Add .refine() ensuring min_block_minutes <= max_block_minutes
- Export type `PlannerSettings = z.infer<typeof plannerSettingsSchema>`
- Export `DEFAULT_PLANNER_SETTINGS` constant with defaults { max_block_minutes: 90, min_block_minutes: 25, buffer_minutes: 10 }
  </action>
