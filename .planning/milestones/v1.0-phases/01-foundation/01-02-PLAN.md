  <action>
    1. Create `app/auth/page.tsx` — Combined sign-up/sign-in page:
       - Client Component ("use client") for tab state management
       - Two tabs: "Sign In" (default) and "Sign Up" — toggle between modes
       - Form fields: email (type="email", required), password (type="password", required, minLength=6)
       - Sign Up form calls `signup` Server Action, Sign In form calls `signin` Server Action
       - Inline error display below form (NOT toast/modal — per user decision in CONTEXT.md)
       - Use URL search params for error messages (Server Actions redirect with ?error=...)
       - Clean Tailwind styling: centered card, max-w-md, clean inputs, submit button
       - Show "Account deleted" message if URL has ?message=deleted

    2. Create `app/auth/actions.ts` — Server Actions ("use server"):
       - `signup(formData: FormData)`:
         - Extract email and password from formData
         - Call `supabase.auth.signUp({ email, password })`
         - On error: redirect to `/auth?error={message}`
         - On success: redirect to `/onboarding`
       - `signin(formData: FormData)`:
         - Extract email and password from formData
         - Call `supabase.auth.signInWithPassword({ email, password })`
         - On error: redirect to `/auth?error={message}`
         - On success: redirect to `/dashboard`
       - `signout()`:
         - Call `supabase.auth.signOut()`
         - Redirect to `/auth`
       - Use `createClient()` from `lib/supabase/server.ts` in all actions
       - Use `redirect()` from `next/navigation` for redirects

    3. Create `app/auth/callback/route.ts` — Auth callback handler:
       - Export GET handler
       - Extract `code` from URL searchParams
       - If code exists: call `supabase.auth.exchangeCodeForSession(code)`
       - Redirect to `/dashboard` on success, `/auth?error=callback-error` on failure
       - This handles email confirmation callbacks (even though disabled for MVP, good to have)
  </action>

  </verify>
