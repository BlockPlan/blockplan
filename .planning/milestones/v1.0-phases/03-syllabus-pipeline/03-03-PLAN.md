  <action>
    Create `app/syllabi/review/actions.ts`:
    - 'use server' directive at top
    - Export `confirmSyllabusItems(courseId: string, items: ParsedItem[]): Promise<{ success: boolean; count?: number; error?: string }>`
    - Authenticate with `supabase.auth.getUser()` — return error if no user
    - Verify courseId belongs to user: query `courses` table with `.eq('id', courseId).eq('user_id', user.id).single()` — return error if not found
    - Validate items array is non-empty — return error if empty
    - Map ParsedItem[] to task insert rows:
      - `user_id`: user.id
      - `course_id`: courseId
      - `title`: item.title
      - `type`: item.type
      - `status`: 'todo'
      - `due_date`: item.dueDate (ISO string or null)
      - `estimated_minutes`: item.estimatedMinutes ?? DEFAULT_MINUTES[item.type] ?? 60
      - `points`: item.points ?? null
      - `weight`: item.weight ?? null
      - `needs_review`: item.needsReview
    - Call `supabase.from('tasks').insert(inserts)`
    - On error: return `{ success: false, error: error.message }`
    - On success: return `{ success: true, count: inserts.length }`
    - Import DEFAULT_MINUTES from `@/lib/validations/task`
    - Import ParsedItem from `@/lib/syllabus/types`
  </action>

  <action>
    Create `app/syllabi/review/page.tsx`:

    **Server Component wrapper (top of file):**
    - Read `course_id` from searchParams
    - If no course_id, redirect to `/syllabi/upload`
    - Authenticate with `getUser()` — redirect to `/auth` if not authenticated
    - Verify course belongs to user — redirect to `/syllabi/upload` if not found
    - Query course name for display
    - Render the client component `ReviewScreen` with courseId and courseName props

    **ReviewScreen client component ('use client'):**

    State:
    - `items: ParsedItem[]` — initialized from sessionStorage `parsedItems-{courseId}` on mount (useEffect). If sessionStorage is empty, show "No items to review" with link back to upload.
    - `editingId: string | null` — which item is currently being edited (null = none)
    - `isConfirming: boolean` — loading state for Confirm All
    - `showAddForm: boolean` — toggle for add-new-item form

    Item list display:
    - Render items as a list/table (Claude's discretion on layout — cards or table rows)
    - Each item shows: title, type badge (color-coded like TaskList), due date (formatted or "No date"), estimated minutes, confidence indicator
    - Items with `needsReview: true` get a yellow/amber warning badge "Needs review"
    - Items sorted by dueDate (nulls last), matching the merge output

    Edit functionality (SYLL-08):
    - Click edit icon on an item → set editingId to that item's id
    - Show inline edit form (or modal) with fields: title (text input), type (select: assignment/exam/reading/other), due_date (date input), estimated_minutes (number input)
    - Save button: update the item in the items array state, clear editingId
    - Cancel button: clear editingId without saving
    - All edits are client-side only — no DB writes

    Delete functionality (SYLL-09):
    - Click delete icon on an item → remove from items array state
    - Optionally show a brief confirmation or use optimistic delete (Claude's discretion)
    - Client-side only — no DB writes

    Add functionality (SYLL-10):
    - "Add Item" button toggles showAddForm
    - Form with same fields as edit: title, type, due_date, estimated_minutes
    - On submit: create new ParsedItem with `id: crypto.randomUUID()`, `source: 'user-added'`, `needsReview: false`, `confidence: 'high'`, add to items array
    - Clear form and hide

    Confirm All button:
    - Disabled when items array is empty or isConfirming is true
    - Shows count: "Confirm {items.length} items"
    - onClick: set isConfirming true, call `confirmSyllabusItems(courseId, items)` Server Action
    - On success: clear sessionStorage key, `router.push(\`/tasks?course_id=${courseId}\`)`
    - On error: set isConfirming false, show error message

    Empty state:
    - If items array becomes empty (all deleted): show message "All items removed. Add items manually or go back to upload."
    - Link to `/syllabi/upload` to re-upload
    - Link to `/tasks` for manual entry

    Navigation:
    - "Back to Upload" link in header area
    - Page title: "Review Syllabus — {courseName}"

    Styling: Tailwind, consistent with existing /tasks page styling. Use similar badge patterns from TaskList.
