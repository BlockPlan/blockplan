<task type="auto">
  <name>Task 2: Build PDF extraction and rule-based parser</name>
  <files>
    lib/syllabus/extract.ts
    lib/syllabus/parser-rule-based.ts
  </files>
  <action>
    Create `lib/syllabus/extract.ts`:
    - Export `extractSyllabusText(storagePath: string): Promise<ExtractionResult>`
    - Use `createClient()` from `@/lib/supabase/server` to download PDF from `syllabi` bucket
    - Convert Blob to ArrayBuffer to Uint8Array
    - Call `getDocumentProxy(uint8)` then `extractText(pdf, { mergePages: true })`
    - Handle text being string or string[] (join with '\n' if array)
    - Set `isEmpty: true` if trimmed text length < 50 characters
    - Throw descriptive errors for download failures

    Create `lib/syllabus/parser-rule-based.ts`:
    - Export `parseWithRules(text: string, termStartDate: Date): ParsedItem[]`
    - Define keyword regex constants:
      - ASSIGNMENT_KEYWORDS: homework, assignment, quiz, project, paper, essay, report, lab, worksheet
      - EXAM_KEYWORDS: exam, midterm, final, test (but NOT "test" when preceded by "pre-" or followed by common non-exam words)
      - READING_KEYWORDS: read, reading, chapter, ch., pages, pp.
      - POINTS_PATTERN: captures digits before pts/points
      - WEIGHT_PATTERN: captures digits before %
    - Split text by newlines, filter lines shorter than 5 or longer than 500 chars
    - For each line: detect type via keywords, extract dates with `chrono.parse(line, { instant: termStartDate })` (MUST pass instant), extract points/weight
    - Set `needsReview: true` when no date found (confidence = 'low')
    - Set `needsReview: false` when date found (confidence = 'high')
    - Generate `id` with `crypto.randomUUID()` for each item
    - Use DEFAULT_MINUTES from `@/lib/validations/task` for estimatedMinutes defaults
    - `cleanTitle` function: remove date text, points text, and excess whitespace from the line to produce a clean title
  </action>
  <verify>
    <automated>npx tsc --noEmit --pretty 2>&1 | head -20</automated>
    <manual>Review regex patterns cover common BYU-Idaho syllabus formats</manual>
  </verify>
  <done>extractSyllabusText downloads from Supabase Storage and returns ExtractionResult. parseWithRules identifies assignments, exams, and readings with correct date resolution via chrono-node.</done>
</task>

<task type="auto">
  <name>Task 3: Build LLM parser and merge logic</name>
  <files>
    lib/syllabus/parser-llm.ts
    lib/syllabus/parser-merge.ts
  </files>
  <action>
    Create `lib/syllabus/parser-llm.ts`:
    - Export `parseWithLLM(text: string, termContext: string): Promise<ParsedItem[]>`
    - Feature gate: if `!process.env.OPENAI_API_KEY`, return empty array immediately
    - Truncate text to 12,000 characters before sending to LLM
    - Use `generateText` from 'ai' with `Output.object({ schema })` and `openai('gpt-4o-mini')` from '@ai-sdk/openai'
    - Define inline Zod schema for the LLM output (array of items with title, type, dueDate, estimatedMinutes, needsReview, confidence)
    - Prompt: "Extract all assignments, exams, and readings from this course syllabus. Term context: {termContext}. For each item extract: title, type, due date (ISO format or null), estimated minutes (or null), and whether parsing was uncertain."
    - Catch `NoObjectGeneratedError` — log and return empty array (fall through to rule-based only)
    - Catch all other errors — log and return empty array (never throw)
    - Map LLM results to ParsedItem[] with source='llm', generate UUID ids, set points/weight to null

    Create `lib/syllabus/parser-merge.ts`:
    - Export `mergeParserResults(ruleItems: ParsedItem[], llmItems: ParsedItem[]): ParsedItem[]`
    - If either array is empty, return the other
    - Deduplication logic: two items are duplicates if they have the same type AND their due dates are within 1 day of each other AND their titles share >60% of words (case-insensitive word overlap)
    - When duplicates found: keep the LLM version (higher confidence), discard the rule-based version
    - Return combined deduplicated array sorted by dueDate (nulls last)
    - Helper function `wordOverlap(a: string, b: string): number` — returns fraction of shared words
  </action>

  </verify>
